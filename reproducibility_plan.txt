# Reproducibility Flags Implementation Plan

> For Devin: execute this plan third, after `frontend_migration_backend_plan.txt`
> and `protocol_import_plan.txt` are committed and tests pass. REQUIRED

> superpowers:executing-plans to implement this task-by-task.

**Goal:** Flag logged protocol-volume mismatches so the lab notebook records what
the researcher said and whether it matches the active protocol step.

**Architecture:** Add a pure backend comparison module that reads the current
step's `parameters["volume_ul"]` and compares it against the volume in
`Command.log_text`. Thread the optional `flag` through log events, SQLite
persistence, `/api/state`, `/api/log`, and the FrontendTest notebook renderer.

**Tech Stack:** FastAPI, Pydantic v2, SQLite with JSON stored as text, vanilla
FrontendTest JS/CSS, pytest.

## Global Constraints

- This plan consumes the FrontendTest serving, shared `FrontendTest/app.js`, and
  `GET /api/log` contract from `frontend_migration_backend_plan.txt`.
- Do not modify old `frontend/` files.
- Do not change the locked ingest spine.
- Do not add a new outer websocket event type.
- Do not add a new `command_result.kind` for reproducibility. Use optional `flag`
  fields on existing `log_entry` and `log_update` payloads.
- The LLM must never compute or mutate reproducibility state.
- Keep v1 scoped to `volume_ul` only. Known v1 gap: a logged volume that matches but
  names a DIFFERENT reagent (e.g. "200 uL ethanol" at a lysis-buffer step) still reads
  as a match — `✓` means "volume matches," NOT "fully verified." Reagent/temp checks deferred.
- Persist the flag because it is part of the lab record.
- Preserve existing log text exactly; warnings never rewrite what was logged.
- Treat `flag` as optional in every renderer.
- VISIBLE flag badges use the display symbols `✓` (match) / `⚠` (mismatch), consistent
  with the project's "pretty symbols allowed in display text" rule (the UI already shows
  `µL`/`°C`). Status must never be symbol-ONLY: a mismatch row also gets the `flagged`
  class and an ASCII screen-reader announcement (keep the announced/aria text symbol-free).
- Run `python -m pytest -q` before commit.

## Task 1: Add the Pure Reproducibility Checker

**Files:**

**Interfaces:**
- Produces `check(step_parameters: dict[str, Any], log_text: str) -> dict[str, Any] | None`.

**Flag shape:**

```python
{
    "parameter": "volume_ul",
    "expected": 200,
    "logged": 250,
    "unit": "uL",
    "status": "ok" | "mismatch",
}
```

**Steps:**

- [ ] **Step 1: Write failing checker tests**

Create `tests/test_reproducibility.py`:

```python
from backend.reproducibility import check


def test_check_returns_none_without_expected_volume():
    assert check({}, "added 200 uL lysis buffer") is None
    assert check({"reagent": "lysis buffer"}, "added 200 uL lysis buffer") is None


def test_check_returns_none_without_logged_volume():
    assert check({"volume_ul": 200}, "pellet looks cloudy") is None


def test_check_accepts_ascii_and_spelled_units():
    assert check({"volume_ul": 200}, "added 200 uL lysis buffer") == {
        "parameter": "volume_ul",
        "expected": 200,
        "logged": 200,
        "unit": "uL",
        "status": "ok",
    }
    assert check({"volume_ul": 200}, "added 200 microliters lysis buffer")["status"] == "ok"
    assert check({"volume_ul": 200}, "added 200 microlitres lysis buffer")["status"] == "ok"


def test_check_flags_mismatch():
    assert check({"volume_ul": 200}, "added 250 uL lysis buffer") == {
        "parameter": "volume_ul",
        "expected": 200,
        "logged": 250,
        "unit": "uL",
        "status": "mismatch",
    }


def test_check_preserves_decimal_values():
    flag = check({"volume_ul": 1.5}, "added 1.25 uL template")
    assert flag["expected"] == 1.5
    assert flag["logged"] == 1.25
    assert flag["status"] == "mismatch"


def test_check_ignores_non_numeric_expected_volume():
    assert check({"volume_ul": "about 200"}, "added 200 uL lysis buffer") is None
```

```javascript
renderLogFlag(flag) {
  if (!flag) return "";
  if (flag.status === "ok") {
    return `<div class="log-ok">OK: ${flag.parameter} matched ${flag.expected} ${flag.unit}</div>`;
  }
  if (flag.status === "mismatch") {
    return `<div class="log-flag">Warning: expected ${flag.expected} ${flag.unit}, logged ${flag.logged} ${flag.unit}</div>`;
  }
  return "";
}
```

Escape dynamic text the same way other log text is escaped.
