# Plan: Electronic Lab Notebook ("Otto") — Hands-Free AI Lab Assistant

> **Two docs, one product.** `real haq doc.pdf` is the **north-star spec** (vision, full
> requirements, data models, NFRs). **This file is the 23-hour build plan** — what we actually
> ship and in what order. Where they disagree, the build plan wins for *what to build now*; the
> PDF wins for *where this is going*.

## Context

UC Berkeley AI Hackathon, **Ddoski's Lab Track**, ~23 hours left.

**Product vision (north star):** an Electronic Lab Notebook (ELN) — a voice-driven assistant that
lets a researcher keep their hands on the experiment and their eyes off the keyboard. They speak
naturally — load a protocol, log an observation, start a timer, find a reagent, ask what's next —
and the system transcribes, parses intent, acts, and **updates the screen** in real time. It's a
**generalized** lab tool: any protocol/inventory loaded from data files works, not just DNA
extraction.

**The non-negotiable demo constraint:** the 30-second pitch is **voice-only — hands never touch
the machine.** Input is voice; output is the screen. (No synthesized speech — see below.)

**Decisions locked with the user:**
- **Stack:** Python / FastAPI backend + lightweight web frontend (vanilla JS, no build step).
- **Voice model:** *Arm once, then hands-free.* Press **Start** before the lab begins (no
  streaming until then); after that the session listens continuously (VAD-gated) and a **wake word**
  ("Hey Otto") gates commands. Satisfies "no manual reactivation between utterances" **and**
  protects Deepgram cost — never stream silence for hours.
- **Output = screen only, no TTS.** The assistant does **not talk**; every response is a panel
  update (current step, log row, timer card, inventory card). Removing Aura cuts a round-trip from
  the response path and lowers cost. *Optional:* a short non-speech **chime** on timer expiry (an
  alert tone, not the assistant talking) so it's noticeable when not looking at the screen.
- **Scope:** **MVP-first, file-driven.** Seed protocols + inventory from data files; the data model
  does the generalizing. No authoring UI in v1.

## Where we're going — build tiers (reconciled with the PDF)

The PDF tiers `find`/inventory as *nice-to-have*, but **demo beat 4 needs it** — so the demo set is
indivisible. The tiering below applies to everything *beyond* the five beats.

| Tier | Contents | Gate |
|---|---|---|
| **Must — the 5 demo beats** | `load_protocol`, `next_step` (deterministic), `log_entry`, `start_timer`, `find_inventory`; live STT; the 5 UI panels. **All five beats, including inventory.** | Ship first; the 30-s pitch works at this tier alone |
| **Fast-follow — "feels real," cheap** | SQLite **log persistence** (survives refresh); Deepgram **custom vocabulary** for lab terms; **auto-timers** on steps with a duration; accessibility (ARIA live + visible alerts); a 2nd protocol for generality | Start only once all 5 beats are stable + demo-clean |
| **Stretch — only if ahead** | Reproducibility flags (logged vol vs protocol vol); methods export (DOCX/PDF); OCR protocol import; open-ended Q&A as a single Claude call over the loaded protocol (**no RAG/vector DB**) | Fast-follow complete |
| **Reserve — never sacrificed** | Polish + **≥5 full demo rehearsals** + recorded **backup video** | Always last |

**Explicitly deferred until the 5 beats work** (PDF distractions for a hackathon): TTS, REST API
surface, RAG/embeddings, OCR, exports, reproducibility flags, multi-user, auth. In-memory state
ships first; SQLite is added *only* for the log, as the first fast-follow.

## Architecture

```
Browser (mic + VAD + UI)
  │  webm/opus chunks (MediaRecorder, 250 ms) — forwarded ONLY while VAD says "speech"
  ▼
FastAPI  ── WS /stream ──┐
  ├─ Deepgram live STT (interim shown; speech_final ends an utterance; KeepAlive in silence)
  ├─ Wake-phrase gate  ("hey otto …" → command; else ignore)
  ├─ Intent router  → ONE structured call → validated Pydantic object   ← the "brain"
  │     intent ∈ {load_protocol, next_step, log_entry, start_timer, find_inventory}
  ├─ Deterministic handlers mutate session state (NO RAG; NO spoken output)
  └─ pushes {ui_update} back over the same WS
  ▲
Browser renders: Live transcript · Current step (prev/next dimmed) · Timers · Log feed · Inventory
```

**Why this shape:** the LLM is a fast **NL → structured intent** parser, not a chat partner — one
call per utterance (the PDF's own "most critical design choice"). Handlers are **deterministic**:
`next_step` just advances a cursor over the loaded protocol (no retrieval). The one place the LLM's
reasoning is load-bearing is `log_entry`, where the *payload is the structured parse*
(text / sample_id / step_ref). **No TTS** — every handler returns a UI-update payload.

## The "brain": bulletproof structured intent routing

Do **not** parse free text or markdown JSON. Force a typed object via native structured outputs /
tool use, with a strict `Literal` enum over the 5 intents:

```python
from typing import Literal, Optional
from pydantic import BaseModel

class Command(BaseModel):
    intent: Literal["load_protocol", "next_step", "log_entry", "start_timer", "find_inventory"]
    protocol_name: Optional[str] = None      # load_protocol
    log_text:      Optional[str] = None      # log_entry
    sample_id:     Optional[str] = None      # log_entry
    duration_s:    Optional[int] = None      # start_timer
    timer_label:   Optional[str] = None      # start_timer
    reagent_name:  Optional[str] = None      # find_inventory
```

- **Anthropic:** `client.messages.parse(model=OTTO_MODEL, output_format=Command, ...)` →
  `resp.parsed_output` is a validated `Command`. Structured outputs are supported on
  **`claude-haiku-4-5`** (and Sonnet 4.6 / Opus 4.8 / Fable 5). Prompt-cache the system prompt.
- **Router is provider-abstracted** behind `route(transcript) -> Command`. Default
  **`claude-haiku-4-5`** (low latency/cost). NOTE: `claude-3-5-haiku` is **retired** — use
  `claude-haiku-4-5`. Swappable to GPT-4o-mini / Gemini Flash (both have native structured outputs)
  **if that's what the hackathon credits cover** — same `Command` schema, one file changes.

## Audio stack (de-risked: MediaRecorder first, PCM only if needed)

**Default path — fast to working, no raw-PCM buffering:** one `getUserMedia` `MediaStream`, tapped
twice:
- **`AnalyserNode`** (Web Audio) computes short-window RMS → the **VAD decision** (speech vs silence).
  Cheap, no encoding — this is what gates cost.
- **`MediaRecorder(stream, {mimeType:'audio/webm;codecs=opus'})`** with **`start(250)`** → emits
  ~250 ms encoded blobs via `ondataavailable`. Deepgram's live endpoint accepts containerized
  `audio/webm;codecs=opus` and resamples server-side — no client downsampling. Forward blobs over the
  WS **only while VAD = speech**; **start/stop the recorder per utterance** so each segment is a
  complete webm container (a single header per segment keeps Deepgram happy; KeepAlive holds the
  socket between utterances).

**Optimization (only if latency demands it):** `AudioWorklet` → 16 kHz linear16 PCM. Do **not** start
here — it can burn 2–3 hours on buffer config. Fall back to MediaRecorder the moment it fights you.

---

## Cost-aware voice pipeline

1. **Arm on keypress.** Nothing streams until **Start session**. Stop/idle-timeout closes the socket.
2. **Client VAD gates audio** (AnalyserNode RMS). Recorder only forwards during speech.
3. **KeepAlive holds the socket cheaply** during silence — warm socket, **no billable audio**.
4. **Endpointing closes utterances fast** via `speech_final: true`.
5. **Wake-phrase gate** ("hey otto …") drops side-chatter in a loud hall. *Optional upgrade:* local
   wake word (Picovoice Porcupine) so STT isn't engaged at all until wake. MVP uses transcript-prefix
   spotting (no extra dep).
6. **No TTS spend at all.** Output is the screen.

Net: streamed seconds ≈ seconds of speech. Nova streaming ~$0.006–0.008/min billed per second; a full
demo costs cents.

## Data model (file-driven; PDF's richer schema)

`data/protocols/dna_extraction.yaml` — steps carry `parameters` + optional `duration_s` (auto-timer):
```yaml
protocol:
  id: dna_extraction
  name: DNA Extraction
  aliases: ["dna extraction protocol", "dna extraction"]
  steps:
    - id: 1
      text: "Add 200 µL lysis buffer to sample tube."
      duration_s: null            # null = manual advance
      parameters: { volume_ul: 200, reagent: "lysis buffer" }
    - id: 2
      text: "Incubate at 65 °C for 10 minutes."
      duration_s: 600             # auto-starts a 10-min timer when activated
      parameters: { temp_c: 65 }
```

`data/inventory.csv` — `name, location, quantity_approx, notes` (fuzzy-matched on `find_inventory`):
```
Proteinase K,Freezer 2 shelf B,~4 doses,10 mg/mL aliquots
Lysis buffer,Fridge 1 shelf A,~200 mL,
```

**State:** in-memory `active_protocol`, `current_step_index`, `timers[]`; **log** in-memory *and*
written to **SQLite `notes`** (fast-follow) so it survives refresh (PDF FR-13). Protocols/inventory
load from files at startup.

## Intents / handlers (5 demo beats) — responses are screen updates

| Beat | Utterance | `Command` | Handler → UI update (no speech) |
|---|---|---|---|
| 1 | "Load DNA extraction protocol." | `load_protocol{protocol_name}` | set active → **current-step panel shows "Step 1: add 200 µL lysis buffer."** |
| 5 | "What's next?" | `next_step` | advance cursor → step panel advances (**no RAG**) |
| 2 | "Log: added 200 µL lysis buffer to sample A." | `log_entry{log_text, sample_id}` | structured row in log feed (+ SQLite, fast-follow) |
| 3 | "Start a 10-minute incubation timer." | `start_timer{duration_s, timer_label}` | countdown card; expiry → visible flash (+ optional chime) |
| 4 | "Where's the proteinase K?" | `find_inventory{reagent_name}` | fuzzy match → inventory card "Freezer 2, shelf B, ~4 doses left." |

Open-ended `ask` is **stretch** (single Claude call over the loaded protocol JSON — no embeddings).
Unrecognized/ambiguous → a visible clarification prompt, never a silent failure (PDF FR-23).

## Build order — prove the brain by typing *before* live mic

- **Slice 0 (pre-flight, H0–1).** Confirm credits + reachable models (Deepgram STT; the fast LLM —
  `claude-haiku-4-5` / GPT-4o-mini / Gemini Flash, whatever credits cover). Scaffold FastAPI +
  static page + `.env`.
- **Slice 1 — TYPED end-to-end (H1–4).** A text box → `route()` → handlers → state → all 5 UI panels
  update. **All five beats work by typing, no audio.** Proves the whole product minus the mic and
  de-risks the demo logic first. This is also the live-fallback input channel.
- **Slice 2 — live STT (H4–7).** `MediaStream` → AnalyserNode VAD + MediaRecorder(250ms) webm/opus →
  WS → Deepgram → interim + final → feed `is_final` into the *same proven* `route()`. Add custom
  vocabulary (microliters, proteinase K, lysis buffer, PCR…).
- **Slice 3 — cost controls + wake (H7–9).** Arm-once + VAD gating + KeepAlive + wake-phrase gate;
  mic-status UI (idle / listening / thinking). No TTS to build.
- **Slice 4 — "feels real" (H9–13).** SQLite log persistence; auto-timers; accessibility (ARIA live +
  visible alerts); 2nd protocol; harden the exact 5 utterances + prompts.
- **Slice 5 — polish + fallbacks (H13–17).** UI pass (frontend-design); keep the **typed box +
  push-to-talk** as live fallbacks; WS reconnect + connection-state indicator.
- **Reserve (H17–23).** **≥5 full rehearsals** in a noisy room; tune VAD/wake thresholds + latency;
  record a **backup demo video**; buffer.

(Dropping TTS frees ~1–2 hours vs. the prior order — folded into more Reserve.)

## Risks & fallbacks

- **AudioWorklet/PCM rabbit hole** → default to **MediaRecorder(250ms) webm/opus** (Deepgram resamples
  server-side); only attempt PCM if latency demands it.
- **Mic/VAD flaky in a loud hall** → the **typed box** (Slice 1) + **push-to-talk toggle** are the
  live fallbacks (same `route()` path) — built first, free insurance.
- **Loose/invalid JSON from the router** → native structured outputs + Pydantic `Literal` enum make
  malformed payloads impossible to act on; validation error → visible clarification.
- **Transcription mangles lab terms** → Deepgram custom vocabulary / keyterm prompting.
- **Deepgram WS drops** → auto-reconnect + visible connection state.
- **Latency >2 s** → `claude-haiku-4-5`, trimmed/cached system prompt (no TTS hop to begin with).
- **Live demo fails** → pre-recorded backup video (Reserve).

## Verification

- **Router harness (fast, no audio):** feed the 5 canned transcripts into `route()`; assert each
  returns the right `Command` (e.g. log → `intent="log_entry", log_text="added 200 µL lysis buffer",
  sample_id="A"`; "what's next" → `intent="next_step"`). Run on every prompt tweak. This *is* Slice
  1's acceptance test.
- **End-to-end:** `uvicorn backend.main:app` → Start → speak the 5 lines → each panel updates within
  ~2 s (PDF FR-08), including the timer-expiry alert.
- **Cost check:** Deepgram dashboard shows **streamed seconds ≈ speech seconds**, not wall-clock
  (proves VAD gating + KeepAlive).
- **Persistence (fast-follow):** refresh the page → the log feed reloads from SQLite.
