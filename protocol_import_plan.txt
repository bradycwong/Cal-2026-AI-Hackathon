# Protocol Import Implementation Plan

> For Devin: execute this plan second, after
> `frontend_migration_backend_plan.txt` is committed and tests pass. REQUIRED
> SUB-SKILL: use superpowers:subagent-driven-development or
> superpowers:executing-plans to implement this task-by-task.

**Goal:** Let the user paste a real-world protocol, parse it into the existing
protocol YAML shape, persist it, register it immediately, and show it in the
FrontendTest protocol catalog.

**Architecture:** Protocol import is a deterministic side endpoint with an
LLM-assisted parser and a deterministic fallback. It writes YAML to
`backend/data/protocols/`, re-loads through the canonical protocol loader, and
then reuses the catalog installed by plan 1. The normal voice/typed load spine is
unchanged.

**Tech Stack:** FastAPI, Pydantic v2, PyYAML, Anthropic SDK already used by the
router, vanilla FrontendTest JS, pytest.

## Global Constraints

- This plan consumes the serving and catalog contracts from
  `frontend_migration_backend_plan.txt`.
- Do not re-add or replace `GET /api/protocols`; extend its source data only.
- Do not change the root static mount; FrontendTest is already served from `/`.
- Do not add a new `Command` intent for import.
- Do not add new outer websocket event types.
- Imported protocol status is always `READY`.
- Keep protocol status values uppercase: `READY`, `LOW_REAGENTS`, `ARCHIVED`.
- Server-side API keys only; no vendor key names in `FrontendTest/`.
- Run `python -m pytest -q` before commit.

- [ ] **Step 3: Implement Pydantic parse models**

In `backend/protocol_import.py`, define:

```python
class ParsedStep(BaseModel):
    text: str
    duration_s: int | None = None
    timer_label: str | None = None
    parameters: dict[str, Any] = Field(default_factory=dict)


class ParsedProtocol(BaseModel):
    name: str
    description: str = ""
    aliases: list[str] = Field(default_factory=list)
    steps: list[ParsedStep] = Field(default_factory=list)
```

Do not accept an LLM-supplied protocol id. The backend assigns ids.

- [ ] **Step 4: Implement deterministic fallback**

Fallback behavior:

- Normalize input with `router.normalize_ascii`.
- Split on non-empty lines and numbered markers.
- Keep one action per resulting step.
- Use `router._parse_duration(text)` for `duration_s`.
- Set `timer_label` only when a duration exists.
- Do not invent reagents or numeric parameters in fallback mode.
- Name is `name` argument, first useful line, or `"Imported Protocol"`.

- [ ] **Step 5: Implement LLM parse path**

Mirror the router's Anthropic client conventions. `IMPORT_MODE` values:

- `auto` - default, use LLM when available and fallback on failure.
- `llm` - require LLM parse, return a controlled error if unavailable.
- `deterministic` - always fallback.

The system prompt must instruct the model to:

- Preserve author order.
- Never invent, merge away, or drop steps.
- Use established parameter keys: `volume_ul`, `temp_c`, `speed_g`, `cycles`,
  `reagent`.
- Use ASCII units such as `uL` and `degrees C`.
- Return a one-line description and 1 to 4 lowercase aliases.

- [ ] **Step 6: Implement filesystem helpers**

Add `_slugify(name)`, `_unique_slug(slug, proto_dir, state)`, and `_write_yaml`.
The YAML must match the loader shape:

```python
{
    "protocol": {
        "id": slug,
        "name": final_name,
        "version": "1.0",
        "description": parsed.description,
        "category": "Imported",
        "status": "READY",
        "aliases": aliases,
        "steps": [...],
    }
}
```

Use atomic write via `path.with_suffix(".yaml.tmp")` and `replace()`.

- [ ] **Step 7: Implement orchestrator**

`import_protocol()` should:

1. Reject empty input.
2. Parse via LLM or fallback.
3. Reject zero steps.
4. Assign ids `1..N`.
5. Choose a unique slug without clobbering files or `state.protocols`.
6. Write YAML.
7. Call `state.register_protocol(path)`.
8. Return `(proto, path)`.

- [ ] **Step 4: Add request model and endpoint**

```python
class ProtocolImportIn(BaseModel):
    text: str
    name: str | None = None
```

and:

```python
@app.post("/api/protocols/import")
async def import_protocol_endpoint(body: ProtocolImportIn) -> dict[str, Any]:
    try:
        proto, _ = import_protocol(body.text, body.name, state)
        summary = next(p for p in state.protocol_catalog() if p["id"] == proto.id)
        load_hint = f'Say "Load {proto.name}" to start it.'
        event = protocol_imported_event(
            proto.name,
            proto.id,
            len(proto.steps),
            proto.aliases,
            load_hint,
        )
        await manager.broadcast([event])
        return {"ok": True, "protocol": summary, "load_hint": load_hint}
    except ValueError as exc:
        event = error_event("protocol_import_failed", str(exc), "protocol_import")
        await manager.broadcast([event])
        return {"ok": False, "error": str(exc)}
```

Map internal exceptions to controlled `ok: False` responses. Do not let the UI see
a 500 for malformed prose.
