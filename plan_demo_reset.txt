# Demo Reset Implementation Plan

> For Devin: execute this plan fourth, after the migration, protocol import, and
> reproducibility plans are committed and tests pass. REQUIRED SUB-SKILL: use
> superpowers:subagent-driven-development or superpowers:executing-plans to
> implement this task-by-task.

**Goal:** Add a one-click pre-demo reset that clears the live FrontendTest session
view, protocol cursor, transcript, timers, inventory lookup result, clarification
state, and transient UI state. Persisted lab notes are wiped only when
`LAB_DEMO_MODE=true`.

**Architecture:** Add reset primitives to `SessionState` and `NoteStore`, expose
them through `POST /api/demo/reset`, broadcast a `command_result` kind named
`reset`, and wire a shared FrontendTest reset control through `FrontendTest/app.js`.
Protocol files, imported protocol files, and inventory CSV data are never deleted.

**Tech Stack:** FastAPI, Pydantic v2, SQLite, vanilla FrontendTest JS/CSS, pytest.

## Global Constraints

- This plan consumes `FrontendTest/app.js` from `frontend_migration_backend_plan.txt`.
- Do not modify old `frontend/` files.
- Do not change the locked ingest spine.
- Do not create new outer websocket event types.
- New UI output must be a `command_result` kind named `reset`.
- Do not clear protocol YAML files, imported protocol YAML files, inventory CSV
  rows, or API keys.
- Persisted notes are deleted only when `LAB_DEMO_MODE=true`.
- With `LAB_DEMO_MODE` unset or false, reset clears the operator's screen but
  keeps DB-backed notes; refresh or restart may show retained notes again.
- Reset must clear optional reproducibility flags only as part of clearing log
  entries. Do not add special flag logic.
- Run `python -m pytest -q` before commit.

    self.log = []
    self._log_seq = 0
