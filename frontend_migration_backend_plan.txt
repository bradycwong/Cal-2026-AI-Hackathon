# FrontendTest Backend Migration Implementation Plan

> For Devin: execute this plan first. REQUIRED SUB-SKILL: use
> superpowers:subagent-driven-development or superpowers:executing-plans to
> implement this task-by-task. This plan installs the shared contracts consumed
> by `protocol_import_plan.txt`, `reproducibility_plan.txt`, and
> `plan_demo_reset.txt`.

**Goal:** Make `FrontendTest/` the served live UI and expose the backend snapshots
and websocket payloads it needs.

**Architecture:** FastAPI continues to own the locked ingest spine and websocket
bus. New REST endpoints are read snapshots or deterministic side endpoints. A new
shared `FrontendTest/app.js` hydrates each static mockup page from those contracts
and handles websocket updates.

**Tech Stack:** FastAPI, Pydantic v2, SQLite, PyYAML, vanilla HTML/CSS/JS,
pytest, FastAPI `TestClient`.

## Global Constraints

- Preserve the locked spine: `transcript -> route() -> handle_command() -> broadcast()`.
- Do not add new outer websocket event types. Keep `transcript_update`,
  `command_result`, `timer_update`, and `error`.
- `step_change` remains a `command_result` kind and gains optional tracker fields.
- All new page-data endpoints are same-origin under `/api/*`.
- Serve `FrontendTest/` from `/`; keep old `frontend/` files but stop targeting
  them in tests after this plan.
- Register the root static mount last so `/api/*` and `/ws/*` routes win.
- Protocol status values are uppercase: `READY`, `LOW_REAGENTS`, `ARCHIVED`.
- Inventory status values are lowercase: `ok`, `low`, `critical`, `expiring`.
- Keep vendor keys server-side; no vendor key names in `FrontendTest/`.
- Run `python -m pytest -q` before commit.

## Shared Contracts This Plan Owns

`GET /api/protocols` returns:

```python
{
    "protocols": [
        {
            "id": "dna_extraction",
            "name": "DNA Extraction",
            "description": "Rapid silica-based genomic DNA extraction.",
            "category": "Genomics",
            "status": "READY",
            "est_duration_min": 45,
            "duration_s": 2700,
            "duration_label": "45m",
            "step_count": 5,
            "reagents": ["lysis buffer"],
            "aliases": ["dna extraction"],
        }
    ]
}
```

`POST /api/protocols/{id}/load` returns `{"ok": True, "events": [...]}` and
broadcasts the same events. It builds `Command(intent="load_protocol",
protocol_name=<id>)`, runs `handle_command`, and never calls the router LLM.

`GET /api/inventory` returns `{"items": [...]}` with:

```python
{
    "name": "Lysis Buffer",
    "code": "LYS-1002-B",
    "location": "Cold Room A",
    "category": "Molecular Reagent",
    "quantity_approx": "20 mL",
    "date": "2024-01-15",
    "status": "low",
    "notes": "For DNA extraction",
}
```

`GET /api/log` returns `{"log": [...]}` with log entries including optional
`category`. Later plans may add optional `flag`; this plan should preserve unknown
extra fields in client rendering.

`GET /api/state` keeps the existing shape and enriches `step` with:

```python
{
    "all_steps": [{"id": 1, "text": "Add lysis buffer."}],
    "current_index": 0,
    "protocol_name": "DNA Extraction",
}
```

    if seconds <= 0:

    minutes = max(1, round(seconds / 60))
    if minutes < 60:

    hours, rem = divmod(minutes, 60)
    return f"{hours}h" if rem == 0 else f"{hours}h {rem}m"
